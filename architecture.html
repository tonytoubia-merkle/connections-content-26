The architectureA · 00

Built on Salesforce.
Built differently.

The Tachibana experience runs on the Salesforce platform — Commerce Cloud, Agentforce, Data Cloud, Marketing Cloud Advanced, Personalization, Loyalty Management, Service Cloud — composed through Merkle's Headless 360 foundation and assembled by an agentic build layer. No middleware. Same trust layer.

What follows is the architecture in three strata: the Surface Sarah touches, the Profile the system holds, and the Orchestration that acts on her behalf — with the Einstein Trust Layer governing every read and every action.

Layer one

The Surface.

Where Sarah, the Bantō, and her advisor meet — every channel a different surface, every profile the same.

The places the experience actually happens. The Tachibana storefront, the Bantō she talks to, the console her advisor opens, the email that lands at 8:14 PM. Each surface is a different render of the same customer — fed by the same profile, governed by the same guardrails.

Headless 360
Merkle
The storefront foundation
Composable, brand-skinned React build sitting on Salesforce primitives. Tachibana.com stood up on it; the chrome, the catalog logic, the agent surface, the recompose behavior — all in one foundation, deployable many times.
Agentforce
Salesforce
The Bantō, reasoning
Atlas Reasoning Engine grounded in the customer profile. The Bantō as a configured agent: declared preferences honored, surface recomposed on intent, every reply explainable. Hands off to the human advisor with full provenance.

Salesforce

Personalization

The Data Cloud-native decisioning engine (formerly Marketing Cloud Personalization). Sitemap captures intent in flight. The hero, the recompose tiles, the email subject line — all decisions, not static layouts.
Service Cloud

Interactions SDK

Layer two

The Profile.

What the system knows about Sarah — and how it knows it. One profile. Every signal. Provenance preserved.

The unified customer — not a record, not a CDP segment, but a continually-resolving identity with declared, observed, inferred, partner, and appended signals stacked alongside one another. Each signal tagged with provenance and confidence so the agent knows what it knows, and how it knows it.

Layer three

The Orchestration.

What the system does on her behalf — composed for Sarah, dispatched across channels, governed end-to-end.

Not "described a campaign, generated copy." The agent takes one person's full context and orchestrates a multi-step, multi-channel journey around her goal — not a segment she happened to land in. Email, SMS, CTV, onsite — all composed, not pre-built.

The build

How it was assembled.

The same architecture you've just read could take a quarter. This one took weeks — because the work that used to be the bottleneck (clicks through Setup, manual metadata wrangling, copy-and-paste flow construction) is now the throughput.

Foundation
Headless 360
Merkle's composable Salesforce build. Brand-skinned once, deployable many times. The Tachibana site, the Bantō surface, the clienteling console — all stood up on it.
Build assistant
Claude Code
Agentic build in the IDE. Reads the SFDX project, reasons over metadata, generates Apex / LWC / Flows / Personalization snippets. The configuration layer of Salesforce, expressed as text.
Deployment
Salesforce CLI
sf project deploy start
Programmatic deployment of the full org. Scratch orgs, sandboxes, prod — same CLI, same metadata source format, same audit trail.
Native surface
LWC + Apex
For the pieces that live inside Salesforce — the advisor console, the loyalty actions, the trigger flows — native components. No wrapper. No middleware.
Configuration is the throughput. Not the bottleneck.

The spineA · 05

One trust layer. Every read. Every action.

The Einstein Trust Layer is not a feature on top of the architecture — it is the connective tissue. Every Bantō reply, every clienteling handoff, every composed journey passes through prompt grounding, PII masking, output scoring, and an immutable audit trail. This is why luxury can put it in production.

01 · Prompt grounding
Every model call grounded in the customer profile.
02 · PII masking
Sensitive fields never leave the org boundary.
03 · Output scoring
Toxicity, brand-voice, and policy on every reply.
04 · Audit trail
Provenance preserved — what the agent knew, said, did.
